Custom Subdomain Setup for me-qr.com

This guide provides step-by-step instructions for setting up a custom subdomain with both Cloudflare and non-Cloudflare (premium hosting) DNS configurations.

Cloudflare Setup

For domains managed through Cloudflare DNS

CNAME Record DNS Only

Non-Cloudflare Setup

For cPanel, Plesk, or other hosting providers

CNAME Record TTL: Auto/3600

Target for Both Setups:

px106cl.me-qr.com

1

Log in to Your Cloudflare Account

Go to Cloudflare and log in with your credentials. Select the domain where you want to add the subdomain.

2

Add a Subdomain in Cloudflare

Navigate to the DNS section for your domain within Cloudflare. Click on "Add record" to create a new DNS record.

3

Configure DNS Settings for Subdomain

Type: CNAME
Name: qrcode (or your desired subdomain)
Target: px106cl.me-qr.com
TTL: Auto
Proxy Status: DNS only (‚ö†Ô∏è Important: not proxied)

Critical: Set Proxy Status to DNS only. This ensures Cloudflare is not proxying the traffic and the CNAME record works correctly without interference.

4

Remove the "A" Record (If Needed)

If you already have an existing A record for the subdomain (e.g., qrcode.domain.com), you will need to remove it and create the new CNAME record as mentioned above.

This step ensures that the subdomain correctly points to me-qr.com and not an old or conflicting IP address.

5

Verify and Save Changes

Once the DNS record is set, click Save to apply the changes.

DNS changes can take anywhere from 3 to 24 hours to fully propagate, so be patient.

6

Configure SSL (Automatically Handled)

You don't need to worry about manually setting up SSL for your subdomain, as me-qr.com will automatically configure SSL for your subdomain.

If you decide to upload your own SSL certificate later, make sure it doesn't conflict with the automatic SSL configuration from me-qr.com.

7

Verify DNS Propagation

Use an online tool like DNSChecker to verify that your subdomain (e.g., qrcode.domain.com) is correctly pointing to px106cl.me-qr.com.

8

Set Up on me-qr.com

Go to the me-qr.com/qr-code-generator page and navigate to the User Domain section.

Add your custom domain (e.g., qrcode.domain.com) as instructed. Select whether you want this domain to be applied to all QR codes or specific folders.

After adding the domain, all future QR codes will have links pointing to your custom subdomain.

1

Log in to Your Hosting Provider's Control Panel

Access your hosting provider's dashboard (like cPanel, Plesk, or any custom control panel).

Locate the DNS management or Zone Editor section. This is where you'll manage your domain's DNS records.

2

Add a Subdomain in Your Hosting Control Panel

Find the section where you can create or manage subdomains. This section may be under the "Domains" tab.

Create a new subdomain (e.g., qrcode.domain.com). Make sure you enter the subdomain name correctly.

Example: If your domain is domain.com and you want to create qrcode.domain.com, you'll input "qrcode" as the subdomain name.

3

Modify DNS Records for the Subdomain

After creating the subdomain, navigate to the DNS management section (often called DNS Zone Editor or similar).

Type: CNAME
Name: qrcode (the subdomain you created)
Target/Value: px106cl.me-qr.com
TTL: Auto or 3600 seconds

This ensures that requests to your subdomain (qrcode.domain.com) will be directed to the correct location for QR code generation.

4

Remove Any Existing A Record (if applicable)

If there is an A record already pointing to the subdomain (for example, qrcode.domain.com), remove it. This A record may be pointing to an old IP address.

Ensure there is only one record for the subdomain: the CNAME record.

5

Check for SSL Configuration

Some hosting providers offer SSL certificates automatically for subdomains, while others may require you to configure SSL manually.

me-qr.com will automatically handle SSL for your subdomain, but if you're using a custom SSL certificate, you must ensure that it doesn't interfere with the automatic SSL configuration from me-qr.com.

6

Save DNS Changes

After setting up the CNAME record and removing any conflicting A records, save the changes.

DNS changes may take anywhere from 3 to 24 hours to propagate, depending on your hosting provider and TTL settings.

7

Add the Custom Domain to me-qr.com

Go to me-qr.com/qr-code-generator and navigate to the User Domain section.

Enter the subdomain you created (e.g., qrcode.domain.com).

You may need to choose whether the custom domain will apply to all QR codes or specific folders.

Once you save this, all your QR codes will start using the custom subdomain.

8

Verify DNS Propagation and Test Your Subdomain

Use a DNS checker tool like DNSChecker to verify that your subdomain is properly pointing to px106cl.me-qr.com.

Once DNS has propagated, try visiting your subdomain (e.g., qrcode.domain.com) in a browser to confirm that it correctly redirects to me-qr.com.

DNS Propagation

DNS changes can take anywhere from 3 to 24 hours to fully propagate across the internet. If it doesn't work immediately, wait and test again later.

SSL Certificates

If you manage your SSL certificates manually, be sure the SSL certificate is correctly set up for your subdomain. However, me-qr.com will automatically provide SSL for your custom subdomain, so manual SSL configuration may not be necessary unless you're using your own SSL certificates.

Avoid Overwriting Records

Be cautious when editing DNS records. Ensure that you're not overwriting or accidentally deleting important records that may affect your main domain or email services.

Final Notes

After completing these steps, all QR codes generated through me-qr.com will use the custom subdomain (e.g., qrcode.domain.com) as their URL.

If you remove the domain later, the links in the QR codes will no longer work until you set up a new domain.

Remember to check your QR code before distributing it to make sure the custom subdomain is functioning correctly.
