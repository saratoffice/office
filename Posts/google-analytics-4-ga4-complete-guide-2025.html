Google Analytics 4 - The next generation of web analytics

Google Analytics 4 (GA4) represents a fundamental shift in how we measure digital experiences. Unlike its predecessor Universal Analytics, which was built around sessions and pageviews, GA4 is event-based and designed for a cross-platform world. This guide covers everything you need to know about GA4 in 2025.

With Universal Analytics officially sunset in 2024, GA4 is now the only Google Analytics solution. It uses machine learning to fill gaps in your data, provides more granular control over user privacy, and offers a more flexible reporting structure. Understanding its architecture is essential for anyone working in digital analytics.

Key Concepts in GA4

Events

Everything in GA4 is an event. Every interaction—page view, click, scroll, purchase—is recorded as an event with parameters. This replaces the hit types of Universal Analytics (pageview, event, transaction).

Users

GA4 uses the User ID and Google signals to track users across devices and platforms, giving a more complete picture of the customer journey.

Machine Learning

GA4 automatically surfaces insights like purchase probability, churn probability, and predictive audiences without manual configuration.

Explorations

The Analysis Hub in GA4 allows for deeper custom reporting using techniques like funnel analysis, path exploration, and segment overlap.

GA4 Event Structure

Events in GA4 have a flexible structure. Each event includes:

  • Event name: Describes the action (e.g., page_view, purchase, scroll)
  • Event parameters: Additional data about the event (e.g., page_title, currency, value)
  • User properties: Attributes about the user (e.g., language, preferences)

Automatically Collected Events

GA4 automatically tracks basic interactions without any additional code:

  • page_view, first_visit, session_start
  • user_engagement, scroll, click
  • video_start, video_progress, video_complete
  • file_download, form_start, form_submit
// Example GA4 event tracking gtag('event', 'purchase', { currency: 'USD', value: 45.99, transaction_id: 'T_12345', items: [ { item_id: 'SKU_123', item_name: 'Widget', price: 45.99, quantity: 1 } ] });

Key Reports in GA4

Life Cycle Reports

  • Acquisition: Where users come from (organic search, paid ads, social, direct)
  • Engagement: What users do on your site—pages and screens, events, conversions
  • Monetization: Ecommerce purchases, in-app purchases, ad revenue
  • Retention: How many users return over time—cohort analysis

User Reports

  • Demographics: Age, gender, location, language
  • Tech: Device category, browser, operating system

New Features Unique to GA4

Predictive Metrics

GA4 uses machine learning to generate predictive metrics (requires sufficient data):

  • Purchase probability: Likelihood a user will purchase in the next 7 days
  • Churn probability: Likelihood a user will not return in the next 7 days
  • Predicted revenue: Expected revenue from a user in the next 28 days

Explorations (Analysis Hub)

Five techniques for deep analysis:

  • Free form: Cross-tab reports with custom dimensions and metrics
  • Funnel exploration: Visualize user drop-off between steps
  • Path exploration: See user navigation paths
  • Segment overlap: Compare different user segments
  • Cohort exploration: Analyze user behavior over time

Privacy-First Design

  • Consent mode: Adjusts behavior based on user consent choices
  • Cookieless measurement: Uses modeling to fill gaps when cookies aren't available
  • IP anonymization: Automatically applied

Migrating from Universal Analytics to GA4

If you're still running Universal Analytics (though data collection stopped in 2024), here's what you need to know:

  • Run in parallel: Best practice is to run both properties simultaneously for at least one year to build historical data in GA4.
  • Recreate goals: UA goals don't automatically migrate—you need to mark events as conversions in GA4.
  • Redefine audiences: Audience definitions work differently in GA4; rebuild them based on events.
  • Update dashboards: Any custom dashboards or reports need to be rebuilt for GA4's data model.
  • Train your team: GA4's interface and terminology are different—invest in training.

GA4 Setup Checklist

  1. Create GA4 property: In Google Analytics, click Admin → Create Property → GA4
  2. Install tracking: Add gtag.js or Google Tag Manager to your site
  3. Verify data: Use Realtime report to confirm data collection
  4. Configure events: Set up enhanced measurement events or custom events as needed
  5. Mark conversions: Identify key events and mark them as conversions
  6. Set up audiences: Create audiences for remarketing and analysis
  7. Link to Google Ads: If you run ads, link your GA4 property to Google Ads
  8. Configure privacy: Set up consent mode and data retention settings

Important: Universal Analytics stopped processing new hits in July 2024. If you haven't migrated to GA4 yet, you've lost data collection in the interim. Activate GA4 immediately and use historical UA data for comparisons only.

Limitations to Be Aware Of

  • No unsampled reports in standard: Explorations have limits on cardinality and sampling may apply.
  • Data retention: User-level data is retained for only 2 or 14 months (you choose).
  • Learning curve: The interface is completely different from Universal Analytics.
  • Historical data: You cannot import UA data into GA4; it starts fresh.
  • Custom reporting: Requires learning explorations or using Looker Studio.
  • Hit limits: 1 million events per month for free tier; additional events cost.

GA4 Best Practices for 2025

  1. Plan your event tracking: Document all events you need before implementation.
  2. Use consistent naming: Follow naming conventions (snake_case or camelCase) for event parameters.
  3. Limit custom dimensions: Stay within the 50 custom dimensions per property limit.
  4. Create key events early: Mark conversions as soon as you set up tracking.
  5. Link to BigQuery: For enterprise needs, set up BigQuery export to own your raw data.
  6. Set up alerts: Configure custom alerts for significant data drops.
  7. Use Google Tag Manager: Easier to manage GA4 tags and updates.
  8. Regularly audit: Use the DebugView and Realtime reports to validate tracking.

GA4 and Google Ads Integration

GA4 offers deeper integration with Google Ads than Universal Analytics did:

  • Bidirectional syncing: Audiences created in GA4 appear in Google Ads automatically.
  • Conversion tracking: Use GA4 conversions in Google Ads campaigns.
  • Enhanced attribution: Data-driven attribution across channels.
  • Predictive audiences: Use GA4's machine learning to create audiences like "likely to purchase in next 7 days" for Google Ads targeting.

Conclusion

Google Analytics 4 is not just an upgrade—it's a completely new analytics paradigm. Built for a privacy-first, cross-platform world, GA4 offers:

  • ✓ Event-based data model for maximum flexibility
  • ✓ Machine learning insights without configuration
  • ✓ Unified web and app measurement
  • ✓ Privacy-centric design with consent mode
  • ✓ Deeper Google Ads integration

The learning curve is steep, but the capabilities justify the investment. Start exploring GA4 today—your future self (and your marketing ROI) will thank you.
