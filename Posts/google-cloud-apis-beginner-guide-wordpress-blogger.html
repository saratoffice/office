üß† What Are API Keys and OAuth 2.0 Client IDs?

‚ñ∂Ô∏è API Keys

An API key is a unique identifier used to authenticate requests to Google APIs. It allows your website or app to access Google services like:

  • Maps
  • YouTube Data
  • Google Analytics
  • Cloud Storage

Each API key can be restricted to specific services or HTTP referrers (like your domain), increasing security.

‚ñ∂Ô∏è OAuth 2.0 Client IDs

These are used for secure user authentication‚Äîespecially when your website or app accesses a user's private Google data (like Google Drive or YouTube uploads). OAuth enables:

  • Google Sign-In on your website
  • Accessing a user's files with their permission

üîß APIs You've Enabled ‚Äì and Their Uses

Maps JavaScript API

Embed interactive maps on your site ‚Äì show locations, markers, and routes. Perfect for office locations or stores.

Geocoding API

Convert addresses into geographic coordinates (and vice versa). Use for location search or directions.

Maps Elevation API

Provides elevation data ‚Äì useful for travel or hiking apps.

Maps Embed API

Embed a map as an <iframe> ‚Äì easier alternative to JavaScript API.

YouTube Data API v3

Display playlists, videos, and channel info on your site. Automate content from YouTube to your blog.

YouTube Reporting API

Get bulk reports about your YouTube channel (views, earnings, etc.)

Google Analytics API

Get analytics data programmatically ‚Äì show most visited pages on your site.

Google Drive API

Embed or interact with Google Drive files (Docs, PDFs, Sheets).

Cloud Storage & API

Store and serve files (images, videos, backups) ‚Äì fast-loading assets for WordPress/Blogger.

<script>
  function start() {

‚úÖ Advanced: List Files from Drive Using API

<script src="https://apis.google.com/js/api.js"></script>
<script>
  function start() {
    gapi.client.setApiKey('YOUR_API_KEY');
    gapi.client.load('drive', 'v3', function() {
      var request = gapi.client.drive.files.list({
        q: "'FOLDER_ID' in parents and trashed = false",
        fields: "files(id, name, webViewLink)"
      });
      request.execute(function(response) {
        var files = response.files;
        files.forEach(file => {
          document.write(`<a href="${file.webViewLink}" target="_blank">${file.name}</a><br>`);
        });
      });
    });
  }
  gapi.load('client', start);
</script>

‚òÅÔ∏è Google Cloud Storage API ‚Äî Host and Serve Files

üöÄ Why Use Cloud Storage?

  • High-speed delivery of images, PDFs, videos
  • More control over file access
  • Reduce load on your WordPress server

‚úÖ Step-by-Step: Create a Bucket and Upload Files

1 Go to Google Cloud Console ‚Üí Cloud Storage ‚Üí Create Bucket
2 Name: unique (e.g., saratoffice-assets)
3 Location: Multi-region for best performance
4 Access Control: Uniform
5 Upload files, then make public: Click file ‚Üí Edit Permissions ‚Üí Add allUsers with Storage Object Viewer

‚úÖ Get Public URL

https://storage.googleapis.com/[BUCKET_NAME]/[FILE_NAME]

‚úÖ Use in HTML

<img src="https://storage.googleapis.com/saratoffice-assets/banner.jpg" width="100%" />
<iframe src="https://storage.googleapis.com/saratoffice-assets/doc.pdf" width="640" height="480"></iframe>

‚úÖ WordPress Plugin: WP Stateless

Automatically upload future media to Cloud Storage ‚Äì connect with Service Account JSON key.

üîê Security Best Practices

  1. Restrict API Keys ‚Äì Use HTTP Referrer Restrictions to limit API use to your domains: saratoffice.in and saratoffice.blogspot.com
  2. Monitor Usage ‚Äì Go to Google Cloud Console ‚Üí Monitoring ‚Üí Set alerts for high API usage
  3. Billing & Quotas ‚Äì Some APIs have free monthly quotas; exceeding them will charge your billing account
  4. Don't share keys ‚Äì Never expose keys in public repositories or front-end code if not necessary
